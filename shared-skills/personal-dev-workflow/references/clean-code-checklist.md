# 代码审查清单

## 正确性

- [ ] 边界条件是否处理？
- [ ] 空输入/空集合是否安全？
- [ ] 资源是否妥善关闭（文件、临时目录）？
- [ ] 并发/竞态是否考虑？

## 健壮性

- [ ] 异常捕获具体类型（非裸 `except Exception`）
- [ ] 错误信息是否清晰（包含上下文）？
- [ ] 输入是否经过验证（类型、范围、格式）？

## 可维护性

- [ ] 函数是否单一职责（≤20 行核心逻辑）？
- [ ] 命名是否自解释（不看注释能理解）？
- [ ] 是否有重复逻辑可提取？
- [ ] 注释解释「为什么」而非「做什么」？

## 可测试性

- [ ] 依赖是否可注入（而非全局状态）？
- [ ] 副作用是否隔离（文件/网络/时间）？
- [ ] 纯函数是否分离出来？

## 兼容性

- [ ] 路径使用 `pathlib.Path`（非字符串拼接）？
- [ ] 编码显式指定 `utf-8`？
- [ ] Windows/macOS/Linux 行为一致？
- [ ] 控制台输出 ASCII 安全（无 emoji）？

## 性能

- [ ] 是否有不必要的文件 I/O？
- [ ] 循环内是否有重复计算？
- [ ] 大文件/大数据集是否流式处理？

## 类型安全

- [ ] 公共函数有参数和返回类型注解？
- [ ] 使用 `TypedDict`/`dataclass` 替代裸 dict？
- [ ] Optional 值正确标注 `Path | None`？
